# 週次提出率 SKILL

日報の**週次提出率**（その週に提出対象者の80%以上が1回以上投稿したか）を集計し、毎月1日に前月分をDiscordで発表する仕組みです。**複数の日報お部屋（チャンネル）**に対応しています。

---

## 1. 週次提出率とは

- **定義**: 1週間のうち、**提出対象者（休会中除く）**の**80%以上**が、その週に**1回以上**日報を投稿したかどうか。
- **週の区切り**: 毎月を「第1週（1–7日）、第2週（8–14日）、第3週（15–21日）、第4週（22日–月末）」に分けて集計。
- **達成**: 4週とも80%以上なら「4/4週 達成」。

---

## 2. 関連ファイル・スクリプト（002_AI_ 直下）

| ファイル | 役割 |
|----------|------|
| `export_logs.py` | Discordから日報ログを取得し、`nippou_logs/` にログ＋サマリーを保存。提出率の分母は「提出対象者（休会中除く）」で計算。 |
| `report_feb_weekly.py` | 指定月の週ごとの提出率を集計。1チャンネル指定 or 全チャンネル。手動確認・テスト用。 |
| `post_monthly_report.py` | **前月**の週次提出率を全対象チャンネル分まとめ、Discordに1通でアナウンス。毎月1日9時のcron用。 |
| `nippou_logs/` | ログ `{チャンネル名}.txt` とサマリー `{チャンネル名}_サマリー.txt` を保存。ここにある「ログ＋サマリー」のペアが週次提出率の対象になる。 |

---

## 3. 全部屋をまとめて報告できるようにする（初回・定期）

**月次アナウンスで「他部屋も含めてまとめて報告」するには**、対象にしたい全チャンネル分のログとサマリーを `nippou_logs/` に揃える必要があります。

**手順（時間がかかってもよい場合）**:

1. **全部屋いっぺんに取得**  
   ```bash
   cd /Users/kasaimami/002_AI_
   ./run_export_all_rooms.sh
   ```  
   または  
   ```bash
   cd /Users/kasaimami/002_AI_ && source venv/bin/activate && python export_logs.py
   ```  
   - **TEST は付けない**（`TEST=1` にすると1チャンネルだけになる）
   - 14チャンネル分あるため、**10〜30分ほど**かかることがあります。ターミナルを閉れずに完了まで待つ
2. 完了後、`nippou_logs/` に各チャンネルごとに `{チャンネル名}.txt` と `{チャンネル名}_サマリー.txt` が増えていることを確認
3. 以降、`post_monthly_report.py`（毎月1日9時のcron）は、ここにある「ログ＋サマリー」のペアを**自動で全部**集計し、1通のアナウンスにまとめます

※ 毎週月曜9時のcronで `export_logs.py` を回していれば、その時点で取得されているチャンネル分は更新されます。追加した部屋や、まだ1回も取得していない部屋は、上記の「全部屋取得」を**1回**実行しておくと、月次アナウンスに含まれるようになります。

---

## 4. 他のお部屋（チャンネル）を「1つ」追加するには

**条件**: そのお部屋の日報が `export_logs.py` で取得され、`nippou_logs/` に次の2つが揃っていること。

- `{チャンネル名}.txt` … 日報ログ（行形式: `[YYYY-MM-DD HH:MM:SS] 表示名: 内容`）
- `{チャンネル名}_サマリー.txt` … サマリー（中に「提出対象者（休会中除く）: N」の行があること）

**やること**:

1. **export_logs.py** の `CHANNEL_IDS` にそのチャンネルIDを追加し、`CHANNEL_TO_ROLE` にロール名を追加する。
2. **export_logs.py** を実行して、そのチャンネルのログとサマリーを `nippou_logs/` に取得する（月曜9時のcronで自動取得していれば、追加後は次回から自動で取れる）。
3. それだけで、**post_monthly_report.py** は `nippou_logs/` 内の「ログ＋サマリー」ペアを自動で検出するため、**新しく追加したお部屋も月次アナウンスに含まれる**。別設定は不要。

※ サマリーに「提出対象者（休会中除く）」が無いチャンネル（ロール未設定など）は、週次提出率の集計からはスキップされ、アナウンスには載りません。

---

## 5. 月次アナウンス（毎月1日9時）

- **実行**: `post_monthly_report.py` を**前月**のデータで実行。
- **cron例**: `0 9 1 * * cd /Users/kasaimami/002_AI_ && source venv/bin/activate && python post_monthly_report.py`
- **投稿先**: スクリプト先頭の `MONTHLY_ANNOUNCE_CHANNEL_ID` で指定したDiscordチャンネルに1通だけ投稿。
- **Botトークン**: 環境変数 `DISCORD_TOKEN` か、同じフォルダの `export_logs.py` 内の `DISCORD_TOKEN` を自動参照。

---

## 6. 手動で「〇月の週次提出率」を確認したいとき

- **1チャンネルだけ**:  
  `cd /Users/kasaimami/002_AI_ && source venv/bin/activate && python report_feb_weekly.py nippou_logs/❤️日報：レッド.txt`  
  （例: レッド。他の部屋はファイル名を変える）
- **指定月**: 第2引数で人数を渡すか、サマリーから自動で「提出対象者」を読む。任意月は `report_feb_weekly.py` の `--month` は無いので、必要なら `post_monthly_report.py --month 2026-02 --dry-run` で前月指定して内容だけ確認できる。
- **投稿せず内容だけ見る**: `python post_monthly_report.py --month 2026-02 --dry-run`

## 7. アナウンス文のイメージ

```
【2026年2月 週次提出率のお知らせ】

今月の週次提出率（提出対象者の80%以上がその週に1回以上投稿）の達成状況です。

・**❤️日報：レッド**: 4/4週 ✅ 達成
・**ホワイトの部屋**: 3/4週 ⚠️ 達成（あと1週で達成）

※ 提出対象者数は当月時点のサマリーに基づいています。
※ 休会中の方は提出対象から除いています。
```

---

## 8. 注意・補足

- 提出率の分母は**サマリーの「提出対象者（休会中除く）」**。卒業生はサマリー段階で除外済み。休会中は分母から除くので、休会中だけ未提出なら提出率100%になる。
- ログは `export_logs.py` の取得範囲（2月以降 or 差分）のみ。週次提出率は「その期間のログ」からその月の週別に集計する。
- 新しいお部屋を増やしたら、**一度 export_logs.py を実行**してログとサマリーを取っておくと、翌月からの月次アナウンスに自動で含まれます。
