# 日報「2週間以上未提出」の特定

日報ログとサマリーから、**今日から2週間（14日）以上提出していない人**と**一度も提出していない人**を抽出する。

## データの場所

| 種類 | 場所 | 説明 |
|------|------|------|
| ログ | `nippou_logs/{チャンネル名}.txt` | `[YYYY-MM-DD HH:MM:SS] 名前: 内容` 形式 |
| サマリー | `nippou_logs/{チャンネル名}_サマリー.txt` | 在籍メンバー・未提出者・卒業生（対象外） |
| 卒業生 | `nippou_logs/卒業生リスト.txt` | 1行1名。#はコメント。名前は `_` より前で一致 |

## 手順

1. **対象ログを決める**  
   ユーザーがチャンネルやファイルを指定していればそれを使う。未指定なら `nippou_logs/` 内の `.txt`（`_サマリー` で終わらないもの）を一覧し、選んでもらう。

2. **ログをパースする**  
   `[YYYY-MM-DD HH:MM:SS] 名前: 内容` から「名前」と日付を抜き出し、**名前ごとの最終提出日**を求める。

3. **除外リストを読む**  
   - 同じフォルダの `{チャンネル名}_サマリー.txt` の「## 卒業生（対象外）」と「## 未提出者のうち休会中」の名前を除外対象にする。  
   - `nippou_logs/卒業生リスト.txt` があれば読み、`_` より前の部分で名前一致する人も除外対象にする。

4. **要リマインド対象を出す**  
   次の2つを合わせ、**除外対象（卒業生・休会中）に含まれる名前は除く**。  
   - ログに1回以上出ている人で、最終提出日が**今日から14日より前**の人  
   - サマリーの「## 未提出者」にいる人（在籍で一度も提出していない人。休会中は除く）

5. **結果を提示する**  
   - **要リマインド**: 名前・最終提出日（未提出なら「未提出」）・経過日数  
   - **提出済み（参考）**: 名前・最終提出日・経過日数  
   必要なら、同じフォルダに `リマインド一覧.html` を生成する手順は「日報リマインド」スキルを参照する。

## 週1回のDiscordリマインド

**2週間以上未提出**の一覧を週1回Discordに投稿するには、同梱の `post_weekly_remind.py` を使う。

- **実行**: `cd /Users/kasaimami/002_AI_ && source venv/bin/activate && python post_weekly_remind.py`
- **dry-run**: `python post_weekly_remind.py --dry-run` で投稿せず内容のみ表示
- **設定**: スクリプト先頭の `REMIND_CHANNEL_ID` に送信先チャンネルIDを指定。トークンは `DISCORD_TOKEN` 環境変数または `export_logs.py` から参照。
- **cron例（毎週月曜 9:00）**:  
  `0 9 * * 1 cd /Users/kasaimami/002_AI_ && source venv/bin/activate && python post_weekly_remind.py`

投稿文は全チャンネル分の要リマインドを1通にまとめる。卒業生・休会中は対象外。

## 週1回の日報担当・MVPチェック（毎週月曜）

**日報担当者が週に1回以上返信しているか**と**MVPを出しているか**を先週分で集計し、毎週月曜にDiscordでアナウンスするには `post_weekly_tanto_check.py` を使う。

- **設定**: `nippou_logs/日報担当リスト.txt` に「チャンネル名=担当者表示名」（複数担当はカンマ区切り）を1行1チャンネルで記載。ログに記録されるDiscord表示名をそのまま書く。
- **判定**: 先週（月〜日）のログで、担当者の投稿数が1回以上なら ✅、0回なら ⚠️。本文に「MVPを発表」「今週のMVP」が含まれる投稿があれば MVP ✅、なければ ⚠️。
- **実行**: `python post_weekly_tanto_check.py`（`--dry-run` で投稿せず表示）
- **cron例（毎週月曜、リマインドのあと）**:  
  `0 9 * * 1 cd /Users/kasaimami/002_AI_ && source venv/bin/activate && python post_weekly_remind.py && python post_weekly_tanto_check.py`

投稿先は `TANTO_ANNOUNCE_CHANNEL_ID`（未設定時は `post_weekly_remind.py` の `REMIND_CHANNEL_ID`）を使用。

## 注意

- ログが古いと全員が対象になる可能性がある旨を伝える。  
- 解析できなかった行はスキップする。  
- サマリーがない場合は、ログから分かる人のみを対象にする。
