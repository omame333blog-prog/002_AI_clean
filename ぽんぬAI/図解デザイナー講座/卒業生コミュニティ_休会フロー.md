# 卒業生コミュニティ（アトリエミー）休会フロー

休会希望者向けの決済リンク作成・ご案内、定期決済の停止、および復帰前リマインドまでの一連の流れをまとめた運用ドキュメントです。

---

## フロー（4ステップ）

| # | やること |
|---|----------|
| **1** | **休会連絡が来る**（会員から休会希望の連絡） |
| **2** | **決済フォームを送る**（休会費 1,000円/月。案内文に決済リンクを入れて**チャットワーク**で送付） |
| **3** | **決済確認後、アトリエ継続課金を停止**（UTAGE・決済代行会社で該当会員の定期課金を停止し、会員に「休会処理完了」の連絡を**チャットワーク**で送る） |
| **4** | **連絡ない場合、5ヶ月後にリマインド**（休会開始から5ヶ月経過時点で「来月より自動復帰」の案内を**チャットワーク**で送る。早期復帰した人は対象外） |

※ 2〜3のあいだ：会員が決済完了 → サンクスページ表示。事務局が入金確認 → 継続課金停止 → 「休会処理完了」の連絡を会員に送る。

## ① 休会決済リンクと決済のご案内

### 1-1. 休会決済リンクの作成

- **利用想定**：卒業生コミュニティ（アトリエミー）の休会制度  
  - 月額 1,000円（税込）× 休会月数（**最大6ヶ月**。2ヶ月・3ヶ月など早期復帰も可能）  
  - または「休会開始月〜終了月」に応じた一括/月次リンクを発行
- **決済システム**：  
  - Stripe / ペイパル / ベリトランス / その他利用中の決済サービスで「休会用」の商品・価格を1つ用意し、**休会専用の決済リンク（Payment Link）** を作成する。
- **リンク管理**：
  - 固定で使う場合：月額1,000円の「休会費」用リンクを1本用意し、案内文で「〇月分」「〇月〜〇月分」など支払い対象を明記する。
  - 個別発行する場合：会員ごとに「休会開始月〜終了月」を指定した都度のリンクを発行する。

**リンク作成時の設定例（Stripe の場合）**

- 商品名：アトリエミー 休会費（月額）
- 金額：1,000円（税込）
- 支払いタイプ：一回払い（月ごと案内する場合）または複数月分をまとめた金額

※実際の決済サービス名・画面に合わせて文言・設定は調整してください。

## ② こはるさん、まるりさん、あすぽんさんの定期決済の停止

休会が確定したら、**通常の月額（20,000円/22,000円）の定期決済を停止**する必要があります。  
決済システムの操作権限を持つ以下の3名で、停止手続きの役割を分担・確認します。

### 2-1. 停止前の確認事項

- [ ] 休会承認が完了している（申請・審査済み）
- [ ] 会員への「休会承認・決済のご案内」を送付済み（B区分の場合は入金確認も済んでいる想定）
- [ ] 休会開始日（〇年〇月1日〜）が確定している

### 2-2. 定期決済停止チェックリスト

| # | 作業内容 | 担当 | 完了日 | 備考 |
|---|----------|------|--------|------|
| 1 | 決済システムにログインし、該当会員の「定期・サブスク」を特定する | こはるさん |  |  |
| 2 | 該当会員の定期決済を「停止」または「解約」する | まるりさん または あすぽんさん |  | ※役割は内部分担に合わせて記入 |
| 3 | 停止完了を事務局（またはこはるさん）に報告する | あすぽんさん |  |  |
| 4 | 事務局：会員に「休会処理完了」の連絡をする | 事務局 |  | LINE/メール |

※ こはるさん・まるりさん・あすぽんさんで、誰が「ログイン」「停止操作」「報告」を行うかは、実際の権限に合わせて上表を書き換えてください。  
※ 1人で複数役でも構いません。重要なのは「定期決済が確実に止まっていること」のダブルチェックです。

## ③ 休会から5ヶ月経過時点でリマインドを送る「休会シナリオ」

休会開始から **5ヶ月経過** した時点でリマインドを送り、**6ヶ月目の自動復帰** に向けた案内をします。

### 3-1. シナリオの考え方

- **トリガー**：休会開始日から「5ヶ月後」のタイミング
- **アクション**：事務局から会員へ「1ヶ月後に復帰となります」のリマインド送信（LINEまたはメール）

### 3-2. 運用パターン（ツール別）

#### パターンA：スプレッドシート＋手動リマインド

1. **休会者一覧シート**を1つ用意する（例：Googleスプレッドシート）。
2. 列の例：  
   `会員名」「休会開始日」「休会終了予定日」「5ヶ月経過日（リマインド日）」「リマインド送付済み（済/未）」
3. 毎週または毎月初めに「5ヶ月経過日」が今週・今月に含まれる会員を抽出し、該当者にリマインド文を送る。

#### パターンB：カレンダーリマインダー

1. 休会処理完了時に、**休会開始日＋5ヶ月** の日付でカレンダーにリマインダーを登録する（Googleカレンダー・Outlook等）。
2. タイトル例：「【リマインド】〇〇さん 休会5ヶ月経過・復帰1ヶ月前」
3. アラームが鳴ったら、下記「リマインド送付テンプレート」を使って送信する。

#### パターンC：Notion / タスクツールの「期日」

1. 休会者用のデータベースに「休会開始日」を登録する。
2. 「リマインド送付日 = 休会開始日 + 5ヶ月」のプロパティを作り、ビューで「今週〜来週」でフィルタして担当者が送信する。

### 3-3. リマインド送付テンプレート（5ヶ月経過時）

---

**【テンプレート】休会5ヶ月経過・復帰前リマインド**

件名：【アトリエミー】1ヶ月後に復帰となります（リマインド）

〇〇様

お世話になっております。アトリエミー事務局です。

〇〇年〇月よりご休会いただいておりました、休会期間がまもなく6ヶ月を迎えます。  
規定に基づき、**〇〇年〇月〇日頃より自動的に通常プラン（月額20,000円/22,000円）へ復帰**となります。

復帰後は、再度クレジットカードの定期決済が開始されます。  
カードの変更等ご希望がございましたら、〇月〇日までにアトリエ生専用LINEまでご連絡ください。

何かご不明点がございましたら、お気軽にご連絡ください。  
復帰後もどうぞよろしくお願いいたします。

---

※ 氏名・休会開始月・復帰月は都度差し替えてください。

---

### 3-4. 休会シナリオ用「5ヶ月経過日」の計算

- 休会開始日が **〇〇年〇月1日** の場合  
  → **5ヶ月経過日** = その日の5ヶ月後（同日または月末で統一する場合は「〇月1日＋5ヶ月 = 〇月1日」で計算）
- 例：2025年2月1日休会開始 → 5ヶ月経過 = 2025年7月1日頃にリマインド送付  
  → 6ヶ月経過 = 2025年8月1日頃に自動復帰

## 運用時の注意

- ① 決済リンクは**休会専用**と分かる名称にし、通常入会・再入会用リンクと混在しないようにする。
- ② 定期決済の停止は、こはるさん・まるりさん・あすぽんさんのうち誰が実行し誰が確認するか、事前に役割を決めておく。
- ③ リマインドは「5ヶ月経過」で送ることで、会員が復帰・カード再課金を忘れないようにする。

必要に応じて、決済サービス名・担当者名・URLを実際の環境に合わせて差し替えてご利用ください。
