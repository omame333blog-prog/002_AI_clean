## noteで使えるMarkdownノウハウまとめ

noteは「一部のMarkdown記法」にだけ対応しており、

- **ショートカット入力**: noteエディタ上で記号を入力すると、その場で変換される  
- **貼り付け変換**: Markdownテキストをそのまま貼り付けると、note形式に変換される  

という2パターンでMarkdownを活用できます。ここでは「noteに貼り付けて使える範囲」に絞ってまとめます。

---

## 基本方針

- **noteが対応している要素だけを使う**
  - 見出し: `##` / `###`
  - 引用: `>`
  - 箇条書き: `-` または `*`（どちらかに統一）
  - 番号付きリスト: `1.`
  - 強調（太字）: `**強調したい文字**`
  - 取り消し線: `~~取り消したい文字~~`
  - コードブロック: 「```」で囲む
  - 区切り線: `---`
- **noteに一度変換されたあとは「もうMarkdownではない」**
  - 変換後の文章をコピーしても、Markdown形式には戻らない
  - Markdown原稿は**別ファイルで保存しておく**のがおすすめ
- **文章生成AI（ChatGPTなど）とはMarkdownでやりとりする**
  - 入力も出力もMarkdownでそろえると、note用の下書き管理がしやすい

---

## 見出し（heading）

- **大見出し（h2）**: `## 見出しタイトル`
- **小見出し（h3）**: `### 見出しタイトル`

noteはh2・h3のみ対応しており、他のレベルは自動的に変換されます。

```markdown
# 記事タイトル(h1)      // h2に変換される
## 大見出し(h2)
### 小見出し(h3)
#### 見出し(h4)       // h3に変換される
##### 見出し(h5)      // h3に変換される
###### 見出し(h6)     // h3に変換される
```

---

## 引用（quote）

- 行頭に `> 半角スペース` をつける
- 複数行の引用は、行ごとに `>` をつける

```markdown
> Markdownとは、テキストを構造的に記述する「マークアップ言語」の1つです。
>
> Markdown記法を使えば、ショートカットキーや記号の入力によって、見出しや強調などの装飾や、戻る・進むなどの操作がしやすくなります。
```

---

## 箇条書きリスト（unordered list）

- 行頭に `- 半角スペース` または `* 半角スペース`
- **`-` と `*` は混在させず、どちらかに統一する**
- インデント（スペース）で階層を表現できる

```markdown
- イヌ科
  - 柴犬
  - 秋田犬
- ネコ科
  - 三毛猫
* ヒト科
```

※`-` と `*` を混ぜると、貼り付け時にうまく変換されないことがあるので注意。

---

## 番号付きリスト（ordered list）

- 行頭に `1. 半角スペース`
- 実際には、**数字は何でもよい**（0や同じ数字でもOK）
  - ただし、表示は必ず `1.` から順番に並ぶ

```markdown
1. お湯を沸かす(90°C程度)
2. カップにインスタントコーヒー小さじ一杯を入れ、140mlのお湯を注ぐ
3. コーヒーを飲む
```

次のように書いても、表示結果は上と同じになります。

```markdown
0. お湯を沸かす(90°C程度)
0. カップにインスタントコーヒー小さじ一杯を入れ、140mlのお湯を注ぐ
7. コーヒーを飲む
```

---

## 強調（strong / 太字）

- `**強調したい文字**` が基本（noteではこちらが安定）
- `__強調__` も使えるが、**前後にスペースがないと変換されない**など挙動がややこしい

```markdown
重要な箇所を**強調**します。
重要な箇所を__強調__します。          // 変換されない
重要な箇所を __強調__ します。        // 変換される
```

**おすすめ方針**

- 実運用では **`**太字**` に統一**する
- `__` は使わない or 特殊な場合だけにする

---

## コードブロック（code block）

- 「```」で囲む
- noteエディタ上で ` ``` ` と入力すると、その時点でコードブロックに変換される

```markdown
```
const name = "note";
```
```

※貼り付け用の原稿としては、**「``` で始めて ``` で閉じる」**というMarkdownの基本ルールを守ればOK。

---

## 取り消し線（strike）

- `~~取り消したい文字~~` で囲む

```markdown
文章中に~~取り消し線~~を入れます
```

---

## 区切り線（horizontal rule）

- 行に `---` を単独で書く
- テーマ・話題の区切りを表すときに便利

```markdown
---
```

---

## 実務でのおすすめワークフロー

- **① 原稿はすべてMarkdownで書く**
  - エディタ or ChatGPTなどのAIでMarkdown形式で管理
- **② noteに貼り付けて自動変換させる**
  - 上述の対応記法だけを使っていれば、ほぼ意図どおりに整形される
- **③ 公開後の微調整はnoteエディタ上で**
  - 行間や改行位置の微修正などは、公開前の最終チェックで対応

---

## 最低限おさえておきたい記法まとめ（チートシート）

```markdown
## 大見出し
### 小見出し

> 引用文です。

- 箇条書き1
- 箇条書き2

1. 手順1
2. 手順2

重要なところを**太字**で強調する。
ミスした箇所を~~取り消し線~~で示す。

---

```

このテンプレをベースに、記事ごとに中身だけ差し替えていけば、note用のMarkdown原稿を効率よく量産できます。
