# YouTube台本作成システム 使い方ガイド

このシステムは「コンテキストエンジニアリング」の考え方に基づいて、AIに意図通りの台本を作らせるための仕組みです。

---

## 全体像

```
【準備フェーズ】コンテキストファイルを埋める（初回のみ）
        ↓
【Phase 1】分析・壁打ち → 要件定義書を作成
        ↓
【Phase 2】プロット・ライティング → 台本を作成
```

> **ポイント**：分析と執筆を別タスクに分けることで、変数を減らし品質を向上させています。

---

## ファイル構成

```
03_Stock/リライトコンテキスト（新）/
├── 0_使い方ガイド.md                  ← 今読んでいるファイル
├── リライトコンテキスト（新）.md       ← 【メイン】AIに与えるコンテキスト
├── 要件定義書テンプレート.md           ← Phase 1の成果物テンプレート
├── レビュー観点チェックリスト.md       ← レビュー時の観点
└── 成功台本サンプル/                   ← 成功台本の全文を保存するフォルダ
    ├── 001_タイトル.md
    └── ...

.cursor/COMMANDS/
├── Phase1_分析・壁打ちプロンプト.md    ← Phase 1用のプロンプト
├── Phase2_プロット・ライティングプロンプト.md  ← Phase 2用のプロンプト
└── プロットレビュープロンプト.md        ← レビュー時のプロンプト
```

### メインファイルの構成

`リライトコンテキスト（新）.md` には以下の5つのPartが含まれています：

| Part | 内容 | 重要度 |
|------|------|--------|
| Part 1 | チャンネルコンテキスト（ターゲット、世界観、話し方） | 【必須】 |
| Part 2 | 台本テンプレート（構成の型） | 【必須】 |
| Part 3 | 台本執筆ルール（文章のルール） | 【必須】 |
| Part 4 | 成功台本サンプル（分析 + 抜粋） | 【推奨】 |
| Part 5 | 面白さの言語化（間、感情曲線、キャラクター） | 【推奨】 |

## 準備フェーズ（初回のみ）

### Step 1: 必須パートを埋める（Part 1〜3）

メインファイル `リライトコンテキスト（新）.md` の以下のパートを記入してください。

| 順番 | パート | 何を書くか | 所要時間目安 |
|------|--------|------------|--------------|
| 1 | Part 1: チャンネルコンテキスト | ターゲット、世界観、話し方 | 15〜30分 |
| 2 | Part 2: 台本テンプレート | 台本の構成（セクション定義） | 20〜40分 |
| 3 | Part 3: 台本執筆ルール | 文章のルール、品質基準 | 15〜30分 |

**ポイント：**
- 最初は完璧を目指さなくてOK
- 使いながら「これも追加したい」と思ったら随時更新
- チャンネルごと、ジャンルごとに別ファイルを作ってもOK

**💡 AIに聞くと埋めやすくなる質問：**

| パート | AIへの質問例 |
|--------|--------------|
| Part 1 | 「私のチャンネルは〇〇について発信しています。ターゲットを明確にするために、どんな質問に答えればいいですか？」 |
| Part 1 | 「以下の動画の台本を見て、私の話し方の特徴（一人称、口調、トーン）を分析してください」 |
| Part 2 | 「〇〇系のYouTube動画で効果的な構成パターンを教えてください」 |
| Part 2 | 「以下の台本を分析して、どんなセクション構成になっているか教えてください」 |
| Part 3 | 「以下の台本を見て、私の文章の特徴（文の長さ、表現の癖、避けている言い回し）を分析してください」 |

---

### Step 2: 推奨パートを埋める（Part 4〜5）

精度を上げたい場合は、以下のパートも記入してください。

| 順番 | パート | 何を書くか | 所要時間目安 |
|------|--------|------------|--------------|
| 4 | Part 4: 成功台本サンプル | 成功した台本の分析 + 抜粋 | 30〜60分 |
| 5 | Part 5: 面白さの言語化 | 間、感情曲線、キャラクターの定義 | 20〜40分 |

**ポイント：**
- これらがあると「自分らしさ」が再現されやすくなる
- 最初はなくてもOK、必須だけで始めてみて、物足りなければ追加

## Phase 1: 分析・壁打ち（企画を固める）

### 処理フロー

```
Step0: コンテキスト確認
    ↓
Step1: 競合台本の入力（「完了」まで待機）
    ↓
Step2: 統合分析（コンテキストを踏まえて）
    ↓
Step3: 差別化戦略の壁打ち
    ↓
Step4: 企画決定 → 要件定義書を出力
      （タイトル・サムネ案も確定）
```

### 実行手順

1. 新しいチャットを開く
2. 以下のファイルをAIに渡して実行する

**渡すファイル：**
- `リライトコンテキスト（新）.md`（記入済みのメインファイル）
- `.cursor/COMMANDS/Phase1_分析・壁打ちプロンプト.md`

3. AIが「コンテキストを確認しました」と返答する
4. 競合台本を1本ずつ貼り付ける
5. すべて貼り終えたら「完了」と入力
6. AIが分析結果を出す
7. 差別化戦略を壁打ちする
8. タイトル・サムネ案を確定
9. **要件定義書がファイルに出力される**

> **ポイント**: このフェーズで要件定義書を作成し、Phase 2に引き継ぎます。コンテキストが長くなっても、要件定義書として保存することで情報が失われません。

---

## Phase 2: プロット・ライティング（執筆する）

### 処理フロー

```
Step1: コンテキスト確認
    ↓
Step2: プロット作成（構成案を複数提示）
    ↓
Step3: プロットレビュー（AIレビュー + 目視確認）
    ↓
Step4: 執筆（ライティングメソッドに沿って）
    ↓
Step5: 文章レビュー
    ↓
Step6: 修正・最終出力
```

### 実行手順

1. **新しいチャット**を開く（Phase 1とは別のチャット推奨）
2. 以下のファイルをAIに渡して実行する

**渡すファイル：**
- `要件定義書_〇〇.md`（Phase 1で作成した要件定義書）
- `リライトコンテキスト（新）.md`（メインファイル）
- `.cursor/COMMANDS/Phase2_プロット・ライティングプロンプト.md`
- （任意）`成功台本サンプル/xxx.md`（文体を再現したい場合）
- （任意）`レビュー観点チェックリスト.md`（詳細レビュー時）

3. AIがコンテキストを確認する
4. **プロット（構成案）を複数提示**される → 1つを選択
5. **プロットレビュー**を実施（AIレビュー + 目視確認）
6. OKなら執筆開始
7. 2000文字ごとに確認しながら執筆
8. **文章レビュー**を実施
9. 修正があればファイルを直接編集
10. 完成！

## レビューの使い方

### プロットレビュー（Step3）

プロット（構成案）の段階で、以下の観点でレビューします：

| 観点 | 内容 |
|------|------|
| 構成・流れ | 場面転換、話者の明確さ、感情の起伏 |
| 設定・整合性 | 関係性、リアリティ、ご都合主義の排除 |
| サムネ・タイトル | 構造、加害者セリフ、オチの隠し |

**レビュー依頼テンプレート：**
```
このプロットをレビューしてください。
@レビュー観点チェックリスト.md
@プロットレビュープロンプト.md

【プロット】
（ここにプロットを貼り付け）
```

### 文章レビュー（Step5）

台本執筆後に、以下の観点でレビューします：

| 観点 | 内容 |
|------|------|
| 重複・冗長 | 同じ主張や表現の繰り返し |
| 映像度・具体性 | 視覚描写、悪役のクズ描写 |
| 説得力・論理性 | 証拠の客観性、反論の余地 |
| 時系列 | 時系列の矛盾 |

**レビュー依頼テンプレート：**
```
この台本をレビューしてください。
@レビュー観点チェックリスト.md

【台本】
（ここに台本を貼り付け or ファイル参照）
```

## 成功台本サンプルの取り扱い方

### 保存場所

**成功台本の全文は `成功台本サンプル/` フォルダに保存してください。**

ファイル名は `001_タイトル.md` のように、連番＋タイトルで命名します。

メインファイルには分析結果と抜粋（500〜1,000文字）のみを記入し、台本全文は別ファイルとして保存します。

---

### なぜ台本全文をメインファイルに貼らないのか

- AIは長すぎるコンテキストだと「何が重要か」がぼやける
- 分析結果の方がパターン学習に効果的
- 複数の成功台本を蓄積しても、メインファイルが肥大化しない

### 推奨する構成

| 保存場所 | 内容 | 文字数目安 |
|----------|------|------------|
| メインファイル（Part 4） | 分析結果 + 代表的な抜粋 | 500〜1,000文字 |
| `成功台本サンプル/` フォルダ | 台本全文 | 1万〜3万文字 |

## よくある質問

### Q: なぜPhase 1とPhase 2を分けるの？

**A: 変数を減らし、品質を向上させるためです。**

- Phase 1は「何を作るか」を決める（企画）
- Phase 2は「どう作るか」を実行する（執筆）
- 別タスクに分けることで、それぞれに集中できる
- 要件定義書として保存することで、コンテキストが長くなっても情報が失われない

### Q: 要件定義書は毎回作るの？

**A: はい、新しい台本ごとに作成します。**

Phase 1の成果物として要件定義書を作成し、Phase 2の入力として使います。

### Q: 競合台本は何本必要？

**A: 1〜3本がおすすめです。**

多すぎるとAIの分析が散漫になります。

### Q: 推奨パート（Part 4, 5）は本当に必要？

**A: なくても台本は作れます。**

ただし「なんか違う」「自分らしさがない」と感じたら、Part 4と5を追加すると改善されます。

### Q: ジャンルが複数ある場合は？

**A: ジャンルごとにメインファイルを分けてください。**

例：
- `リライトコンテキスト_ビジネス教育系.md`
- `リライトコンテキスト_ストーリー系.md`

## トラブルシューティング

### 「なんか違う」台本が出てくる

- **原因1**: コンテキストの記入が抽象的すぎる
  - → 具体例を追加する
- **原因2**: 推奨パート（Part 4, 5）がない
  - → 成功台本サンプルと面白さの言語化を追加する

### AIが構成を守らない

- **原因**: Part 2（台本テンプレート）の記入が曖昧
  - → 各セクションの「目的」「含める要素」を明確に書く

### 競合台本と似すぎてしまう

- **原因**: 差別化戦略が弱い
  - → Phase 1のStep3でもっと具体的な差別化ポイントを決める

### 文体が自分らしくない

- **原因**: 成功台本の全文を参照していない
  - → `成功台本サンプル/` フォルダの全文も一緒にAIに渡す

### 展開が都合よすぎる

- **原因**: レビューが不十分
  - → `レビュー観点チェックリスト.md` を使ってプロットをレビューする
  - → 特に「設定・整合性チェック」を確認

## 推奨ワークフロー

### 初めて使う人

```
1. このガイドを読む（今ここ）
2. メインファイルの Part 1〜3 を埋める
3. Phase 1で企画を固める（要件定義書を作成）
4. Phase 2で台本を執筆
5. 「もっとこうしたい」と思ったら、Part 4と5を追加
6. 成功台本の全文を `成功台本サンプル/` に保存
```

### 慣れてきた人

```
【Phase 1】
1. メインファイルを更新（必要に応じて）
2. 競合台本を用意
3. Phase 1プロンプト実行
4. 要件定義書が出力される

【Phase 2】
5. 新しいチャットを開く
6. 要件定義書 + コンテキストを渡す
7. Phase 2プロンプト実行
8. プロットレビュー → 執筆 → 文章レビュー
9. 台本完成
```

## 最後に

このシステムは「AIに何を知らせた状態で考えさせるか」を設計するものです。

コンテキストが充実するほど、意図通りの台本が出やすくなります。最初は完璧を目指さず、使いながら育てていってください。

2フェーズに分けることで、「企画」と「執筆」それぞれに集中でき、より高品質な台本が作れるようになります。
