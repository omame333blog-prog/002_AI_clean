## 1. 重複・冗長チェック

### チェック項目

- [ ] 同じ主張・結論が繰り返されていないか
- [ ] 同じエピソード・具体例が重複していないか
- [ ] 同じ概念の説明が複数箇所にないか
- [ ] 類似の描写表現が近接していないか（例：「顔面蒼白」と「血の気が引いた」）

### 判定基準

| 種類 | 判定 | 対応 |
|------|------|------|
| 完全重複 | ほぼ同じ内容が2回以上 | 削除または統合 |
| 部分重複 | 同じ話の一部が別箇所にも | 統合を検討 |
| 意味重複 | 表現は違うが伝えたいことが同じ | 1つに絞る |
| 意図的繰り返し | 強調のための反復 | OK（ただし3回以上は要検討） |

### 具体例

```
❌ 悪い例：
「泣き寝入りする必要はない」
「諦めないでほしい」
「必ず、逆転のチャンスはある」
「証拠を集めれば、必ず道は開ける」
→ 4文連続で同じメッセージは冗長

✅ 良い例：
「泣き寝入りする必要はない。
証拠を集め、然るべき場所に訴えれば、必ず逆転のチャンスはある」
→ 1文に統合してインパクトを強める
```

## 2. 構成・流れチェック

### チェック項目

- [ ] 場面転換が明確に示されているか
- [ ] 各セリフの話者と状況が分かるか
- [ ] 感情の起伏（アップダウン）があるか
- [ ] 各パート（起承転結 or 転転転結）の役割が明確か

### 推奨構成パターン

#### パターンA：起承転結

| パート | 役割 | 感情 |
|--------|------|------|
| 起 | 理不尽な現実の提示 | ↓ムカつく |
| 承 | 理不尽の拡大 | ↓↓もっとムカつく |
| 転 | 逆転 | ↑スカッと！ |
| 結 | 余韻とメッセージ | 😊よかったね |

#### パターンB：転転転結（推奨）

| パート | 役割 | 感情 |
|--------|------|------|
| 転① | 小逆転（一度スカッと） | ↑勝った！ |
| 転② | 危機（反撃される） | ↓負けるかも... |
| 転③ | 大逆転（完全勝利）★ピーク | ↑↑今度こそ勝った！！ |
| 結 | 平和な日常 | 😊よかったね |

### 具体例

```
❌ 悪い例：
入社してまだ一週間。毎日がこの調子だった。
「あの…新人の村田さんに何か…？」
週明け、本社から来た監査チームの前で、部長は顔面蒼白になっていた。
→ 場面が飛んでいる、セリフの話者が不明

✅ 良い例：
入社してまだ一週間。毎日がこの調子だった。
だが、週明け。状況は一変した。
本社から来た監査チームのリーダーが、部長に尋ねた。
「佐藤部長。村田さんへの暴言について、お話を伺いたいのですが」
部長は顔面蒼白になっていた。
→ 場面転換を明示、話者を明確化
```

## 3. 設定・整合性チェック

### チェック項目

- [ ] 登場人物の関係性が明確か（同じ会社？別組織？）
- [ ] 設定にリアリティがあるか（実在する役職・部署か）
- [ ] 情報の入手経路が説明されているか
- [ ] 展開に無理がないか（都合が良すぎないか）

### よくある問題パターン

| 問題 | 具体例 | 対策 |
|------|--------|------|
| 関係性が不明 | 「みんな同じ会社？」 | 冒頭で組織構造を示す |
| 設定の非現実性 | 「内部監査室長が自ら潜入」 | 「社長直轄の特命」など理由付け |
| 入手経路の欠如 | 「社長がどこからファイルを？」 | 「人事部のサーバーに記録があった」 |
| ご都合主義 | 「想定済みだ」で全て解決 | 客観的な証拠を提示 |

### 具体例

```
❌ 悪い例：
社長「君が佐藤くんを庇うだろうことは、想定済みだ」
→ 都合が良すぎる、なぜ想定できたのか不明

✅ 良い例：
社長「村田くんから報告を受け、過去の内部通報記録を調べさせた」
「人事部のサーバーに、三年前からの通報メールが残っていた」
「君が三回とも握りつぶした記録がある」
→ 客観的な証拠と入手経路を明示
```

## 4. 映像度・具体性チェック

### チェック項目

- [ ] 視覚的な描写があるか（動作、表情、空間）
- [ ] 五感を使った描写があるか（音、匂い、触覚）
- [ ] キャラクターの行動が具体的か
- [ ] 悪役の「クズ描写」が十分か

### 映像度を高める要素

| 要素 | 具体例 |
|------|--------|
| 動作 | 資料をパラパラめくる → 投げる |
| 効果音 | 「バサッ。紙が床に散らばる」 |
| 空間 | 「周囲のキーボードの音が、ピタリと止まる」 |
| 距離感 | 「佐藤が俺の顔を覗き込む」 |
| 五感 | 「コーヒーとタバコの混じった息が、鼻につく」 |
| 周囲の反応 | 「目が合うと、すぐに逸らされる」 |

### 悪役のクズ描写例

```
✅ 良い例：
だが、佐藤自身はどうか。
午前中はデスクでグラビア雑誌を堂々と読んでいる。
昼は二時間の「接待ランチ」。戻ってきたら部下の粗探し。
自分では一切手を動かさない。
それでいて、成果は全て自分のもの。失敗は全て部下のせい。
→ 具体的な行動で「許せない」感情を喚起
```

## 5. 説得力・論理性チェック

### チェック項目

- [ ] 逆転の根拠が客観的か（主観的な主張だけでないか）
- [ ] 証拠が具体的か（数字、記録、第三者証言）
- [ ] 反論の余地がないか
- [ ] 「それで本当に勝てるのか」と思われないか

### 客観的な証拠の例

| 証拠タイプ | 具体例 | 説得力 |
|------------|--------|--------|
| 録音データ | 暴言の録音30件 | ○ |
| メール記録 | 暴言メール50通 | ◎ |
| 第三者証言 | 目撃証言12名分 | ○ |
| 公式記録 | 内部通報の記録、調査記録の不在 | ◎◎ |
| 被害実態 | 退職者5名、療養中1名 | ◎ |

### 具体例

```
❌ 弱い例：
「あなたのパワハラを全て記録しました」
「懲戒解雇の根拠として十分です」
→ 抽象的、反論されやすい

✅ 強い例：
「三年前、この支店から人事部にメールが届いた」
『佐藤部長から毎日暴言を受けています。助けてください』
「このメールに対し、君は『証拠不十分』と返信した」
「しかし、調査した記録がない。電話一本かけていない」
→ 具体的な文書と記録の不在で反論不可能
```

## 6. サムネ・タイトルチェック

### チェック項目

- [ ] 4-5行構成になっているか
- [ ] 加害者のセリフ「〜w」が入っているか
- [ ] オチを適度に隠しているか（全部見せない）
- [ ] 参考タイトルの構造に沿っているか

### 推奨構成（4-5行）

| 行 | 要素 | 例 |
|---|------|-----|
| 1行目 | 舞台設定 | 支店に新人として配属された俺に |
| 2行目 | 加害者の属性 | 毎日暴言を吐きまくるエリート部長 |
| 3行目 | 加害者セリフ | 「三流大学の使えない奴はクビだw」 |
| 4行目 | 転換点 | 一週間後、俺が本当の名刺を出した結果w |
| (5行目) | オチ（隠す場合） | →部長の顔色が変わり... |

### タイトル構造

```
【スカッと】+ 状況設定 + 加害者セリフ「〜w」+ 転換点 + 結果...
```

例：
```
【スカッと】支店に新人として配属された俺に毎日暴言を吐くエリート部長
「三流大学の使えない奴はクビだw」→一週間後、俺が本当の名刺を出した結果w
```

## 7. 時系列チェック

台本内の時系列に矛盾がないかを確認するためのチェック方法です。

### チェック手順

#### ステップ1：時系列要素の抽出

台本内に登場する以下の要素をすべて抽出し、表形式で整理する：
- 具体的な日付・年号（例：2020年、昨年、3年前）
- 時間的順序を示す表現（例：その後、以前、当時、最初は）
- 年齢や経過年数の記述
- イベントの前後関係

| 行番号 | 時系列表現 | 具体的な時点/期間 | 関連するイベント |
|--------|------------|-------------------|------------------|
| | | | |

#### ステップ2：タイムライン作成

抽出した情報をもとに、古い順に並べた時系列表を作成する。

#### ステップ3：矛盾チェック

以下の観点で矛盾がないか1つずつ検証する：
- [ ] Aが起きた後にBが起きたと書いてあるのに、別の箇所でBが先と読める記述はないか
- [ ] 「〇年前」「〇歳の時」などの計算が合っているか
- [ ] 同じ出来事の時期が、箇所によって異なっていないか
- [ ] 「最初は〜」「その後〜」の流れに逆行する記述はないか

### 矛盾が見つかった場合の報告形式

```
- 矛盾箇所1：行〇〇「〜〜」と行〇〇「〜〜」が矛盾
- 理由：〜〜
- 修正案：〜〜
```

## 8. クイックチェックリスト（最終確認用）

### 必須チェック

- [ ] 重複・冗長箇所を削除/統合したか
- [ ] 場面転換は明確か
- [ ] 転転転結の感情起伏があるか
- [ ] 設定に無理がないか
- [ ] 証拠は客観的か
- [ ] 映像度は十分か
- [ ] サムネ・タイトルは構造に沿っているか

### 最終確認の質問

1. **「ここ飛んでない？」** と思う箇所はないか
2. **「このセリフ誰が言ってるの？」** と思う箇所はないか
3. **「都合良すぎない？」** と思う展開はないか
4. **「本当にそれで勝てる？」** と思う逆転はないか
5. **「もっと許せない描写が欲しい」** と思う悪役はいないか
