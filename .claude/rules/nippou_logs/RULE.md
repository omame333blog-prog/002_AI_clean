# nippou_logs ルール（卒業・休会・未提出の扱い）

## ソース・オブ・トゥルース

- **卒業生（提出義務なし）**: `nippou_logs/卒業生リスト.txt`
- **休会中（提出義務なし）**: `nippou_logs/休会中リスト.txt`

## 卒業生リストの書き方

- 1行1名（`#` 始まりはコメント）
- 基本は「ベース名」でもOKだが、**同名が複数いる場合は以下の優先順で指定する**

| 状況 | 書き方 | 例 |
|------|--------|-----|
| 同名なし | ベース名 | `やすこ` |
| 同名あり・表示名で区別可 | 表示名そのまま | `やすこ_B` |
| 同名あり・表示名が同じ | `表示名 @username` | `ゆか @yuka4059` |

- `表示名 @username` 形式は**最優先で照合**される（同名別人でも確実に区別できる）
- `@` は半角・全角どちらでも可（`aya @aya.0730` / `aya ＠aya.0730` 両方OK）
- Discord username は `@ユーザー名` で確認可（表示名とは異なる）
- 目的: **提出率・未提出・リマインド対象から完全除外**する

## ランクサフィックス（_R/_P/_S/_G/_B）の扱い

メンバーの名前末尾 `_R`/`_P`/`_S`/`_G`/`_B` はランク制度によるもので、途中で変わることがある。

- `get_list_key()` は `_` より前のベース名を返す（例: `つかちゃん_P` → `つかちゃん`）
- `post_weekly_remind.py` はログとサマリーをベース名で突き合わせるため、ランク変動があっても誤検知しない
- 卒業生・休会中リストはベース名で書けば全ランクに一致する（`つかちゃん` と書けば `つかちゃん_R` にも `つかちゃん_P` にも一致）

## 休会中リストの書き方

- 1行1名（`#` 始まりはコメント）
- ベース名でOK（例: `Naco` → `Naco_R` に一致）
- **名前に数字が含まれる場合は `_` より前を全部書く**
  - NG例: `みえみえ` （`みえみえ08060715_R` に一致しない）
  - OK例: `みえみえ08060715` （`みえみえ08060715_R` に一致する）
- 目的:
  - **提出対象者（分母）から除外**
  - サマリーでは `未提出者のうち休会中（リマインド対象外）` にのみ表示

## サマリーの出力仕様（必須）

サマリーは必ず次の見出しで区分すること。**「未提出者」は常に「休会中を除く」と明記し、休会中は別見出しで分ける。**

- `## 未提出者（休会中を除く）` … リマインド・週次提出率の対象
- `## 未提出者のうち休会中（リマインド対象外）` … 該当がある場合のみ
- `## 卒業生（対象外）` … 該当がある場合のみ

※ `post_weekly_remind.py` や週次提出率集計は「未提出者（休会中を除く）」の見出し以下を参照する。卒業生・休会中は対象外。

### カウント

- `提出対象者（休会中除く）` は、在籍メンバーから休会中を除いた人数（週次提出率の分母）
- `未提出者（休会中を除く）` は、提出義務ありの未提出者のみ
