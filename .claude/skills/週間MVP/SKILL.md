# 週間MVPの選定

日報ログから**その週のMVP候補**を抽出し、必要に応じて**MVP決定・Discord発表文・スライド理由・読み上げ台本**まで作る。

単なる「頑張り」評価ではなく、次の3軸を重視する。

- **数字**: 応募数、制作本数、修正回数、枚数、納品数、継続件数など
- **変化の幅**: 先週までできなかったことができるようになったか
- **メンタル面**: 落ち込みから戻した、自分の勝ちパターンを使えた、怖いけど出た、など

## 基本方針

- 候補出しの段階では **3〜5名程度** に絞る
- ユーザーが重視する軸が曖昧なら、**積極的に質問する**
- 数字が弱い人を出す場合は、「メンタルの変化」「前週比較」で出す理由を明確に言語化する
- 連続受賞を避けたい運用なら、直近受賞者かどうかを確認する
- 数字は必ず具体化する。曖昧な場合は「少なくとも」「約」「ログ上では」と書く

## よくある依頼パターン

1. **候補だけ出してほしい**
2. **誰にするか一緒に決めたい**
3. **この人でいくので発表文を作ってほしい**
4. **数字よりメンタルや変化の幅で見てほしい**
5. **月間MVP候補として週ログをまたいで見てほしい**
6. **特定の人名だけ言って発表文を作ってほしい（ログ提示なし）** ← ログなし指名パターン

### ログなし指名パターンの対応（重要）

ユーザーが「◯◯さんの発表文を作って」と人名だけ言ってきた場合：
1. どの**部屋**か確認する（14部屋ある）
2. 参照するログファイルの有無を確認する
3. ログがない場合は「その人の今週のエピソードや頑張りを教えてください」と聞いて、ユーザーが提供する内容から発表文を作る
4. ログがある場合はファイルを探してから作る

> NG: ログを探して見つからなかっただけで終わる
> OK: ログがなければユーザーに内容を聞いて発表文を作り切る

- ログが不足している場合は、勝手に補完せず、不足分の有無を確認する
- **人名だけ指定された場合は、部屋名とログの有無を先に確認する**

## 手順

0. **依頼タイプを判定する（最初に必ずやる）**
   - 人名だけ指定 → 「どの部屋ですか？ログファイルはありますか？なければエピソードを教えてください」と聞く
   - 部屋名＋期間が揃っている → 手順1〜へ進む
   - ログファイルが添付されている → 手順3〜へ進む

1. **対象期間を確認する**
   例: `3/2〜3/9`、`今週`、`2/16〜2/23` など。曖昧なら期間を聞く。

2. **対象ファイルを確認する**
   ユーザーがファイルを指定していればそれを使う。指定がなければどのログを見るかを確認する。

3. **ログに対象日が入っているか確認する**
   例: ユーザーが `2/16〜2/23` と言っても、実ファイルが `2/18` までしかないことがある。
   → 今ある範囲で候補を出すか、不足ログを追加でもらうかを確認する。

4. **候補者の動きを抽出する**
   - 応募数、テスト・トライアル数、受注数、継続化
   - 修正→再提出→納品の流れ
   - 枚数、本数、金額
   - 気持ちの変化、回復の仕方、前向きな切り替え

5. **前週や過去との比較を入れる**
   「その週だけの量」よりも「前まで止まっていた人が動けたか」「前週マイナスだった人が0以上に戻ったか」を重視する。

6. **候補を3〜5名に絞って提示する**
   候補ごとに以下を書く。
   - 数字
   - 変化の幅
   - メンタル面
   - 一言でいうと何タイプの候補か

7. **必要ならユーザーに質問する**
   - 数字重視か、変化幅重視か
   - 連続受賞は避けるか
   - 外部案件を重く見るか、講座内成長を重く見るか
   - 回復ストーリーを出したいか

8. **1名に決まったら発表用の文章を作る**

## 候補提示の型

```
**名前**
- 【数字】応募数、制作本数、納品数、継続数など
- 【変化】前週までとの違い
- 【メンタル】どう立て直したか、怖さをどう超えたか
```

## メンタル変化を重視する時の見方

数字が弱くても候補になりうる変化の例:
- 「自信がない」「腰が引ける」から、初応募・初投稿・初テストまで進んだ
- 落ち込みや停滞から、自分なりの戻し方で前向きに戻った
- 人に相談する、自分の勝ちパターンに戻す、朝活を一旦休んで整える、などの**回復手段を使えた**
- 完璧主義から完了主義に少し移れた
- 比較でしんどくなっていた状態から、自分の軸に戻れた

## 発表文の型

1名に決まったら、必要に応じて次を作る。

### Discord用発表文

- 冒頭に対象部屋のメンション（例: `@レッドのお部屋❤️`、`@ホワイトのお部屋🤍` など）
- `今週のMVPは… @name さんです✨`
- 理由は3つ
- 最後に「おめでとうございます」で締める
- 部屋が不明な場合はユーザーに確認する

### 理由づけのルール

- 必ず具体的な数字を入れる
- 数字がその週の変化にどうつながったかを書く
- 可能なら「前はできなかったこと」「今週できるようになったこと」「今後につながる再現性」を入れる

### 3つのアウトプット

1. **Discord用発表文**
2. **スライド用の理由3つ**
3. **読み上げ台本**

## 注意

- ログにないことは言い切らない
- 応募数や件数は日報に出ている範囲で数える
- 正確に言い切れないときは `少なくとも` `約` `ログ上では` を使う
- 単に「頑張っていた」で終わらせない
- 数字が少ない候補を出すときは、**なぜその人を出すのかを必ず明確化する**
- 過去の受賞履歴が判断に関わる場合は、必要に応じてユーザーに確認する
